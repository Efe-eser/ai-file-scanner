What could happen?
.bat / .cmd
Runs commands. If “download-and-run” is used, it can execute system actions, change registry settings, and create persistence.
.ps1
PowerShell scripts can download and execute payloads, steal credentials, and bypass system policies when misused.
.vbs / .js / .scr
Scripts may run via Windows Script Host. Social engineering (“double‑click”) is a common way these get executed.
.exe / .dll / .msi
Directly executable binaries can carry trojans/ransomware, abuse DLL sideloading, and spoof or misuse digital signatures.
.docm / .xlsm
Macro-enabled Office files can run code when “Enable Content” is clicked, which is a frequent infection vector.
.zip / .rar / .7z
Archives can hide executables, trigger decompression bombs, or exploit unsafe extraction paths (e.g., zip‑slip).
.jpeg / .png
Images are not normally executable, but malformed files can exploit viewer/parser bugs or hide payloads (steganography). Risk is usually lower than executables.
Where do my files go?
Temporary processing
Your file is uploaded to our server (e.g., an EC2 instance) only to generate this scan report. After the report is produced, the uploaded file is deleted from server storage.
What we send to third parties
VirusTotal is queried using the file’s SHA‑256 hash (not the raw file). If you click AI Analysis, the file content may be sent to the AI provider to generate a second-opinion explanation.
Important
This is a security demo and does not guarantee a file is safe. Avoid running unknown files, and never enable macros from untrusted sources.
